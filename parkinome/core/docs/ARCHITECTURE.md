# Parkinome Core Architecture

## Project Layout

- `src/`: runtime C sources (`server.c`, `predictor.c`, `model.c`, `json_io.c`)
- `include/`: public and internal headers (`predict.h`, `predictor.h`, `json_io.h`)
- `web/`: browser UI (`index.html`) and static web assets
- `data/`: sample JSON payloads and local data files
- `docs/`: project docs
- `legacy/`: archived modules not used in current runtime path
- `build/`: object files generated by `make`

## Active Runtime Modules

- `src/server.c`: HTTP server and route handling.
- `src/predictor.c` + `include/predictor.h`: unified JSON parsing (`cJSON`) and prediction adapter.
- `src/model.c` + `include/predict.h`: risk model and prediction API.
- `src/json_io.c` + `include/json_io.h`: file read utility for serving `web/index.html`.

## Legacy Modules

`legacy/` stores non-runtime or historical code paths:

- `features.c`
- `validate.c`
- `storage.c` / `storage.h`
- `storage_sqlite.c` / `storage_sqlite.h`
- `logger.c` / `logger.h`
- `model_weights.h`

These files are preserved for reference and possible future reintegration, but are not part of the current HTTP prediction path.

## Request Flow

1. Client sends HTTP request to `src/server.c`.
2. `src/server.c` dispatches route (`/`, `/predict`, `/me`, `/save`, `/history`).
3. `src/predictor.c` parses request JSON and calls `parkinome_predict(...)`.
4. `src/model.c` computes risk outputs.
5. `src/server.c` returns JSON or HTML response.
